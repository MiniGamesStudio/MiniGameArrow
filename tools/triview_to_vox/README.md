# 三视图转 MagicaVoxel 体素模型工具

将模型的三视图（正视图、右视图、顶视图）自动转换为 MagicaVoxel 的 `.vox` 文件。

## 安装

```bash
# 1. 安装 Python (3.8+)
#    下载地址: https://www.python.org/downloads/

# 2. 安装依赖
pip install -r requirements.txt
```

## 使用方法

### 基本用法

```bash
python triview_to_vox.py --front 正视图.png --right 右视图.png --top 顶视图.png -o model.vox
```

### 完整参数

```bash
python triview_to_vox.py \
    --front front.png \      # 正视图 (必填)
    --right right.png \      # 右视图 (必填)
    --top top.png \          # 顶视图 (必填)
    -o model.vox \           # 输出文件 (默认: model.vox)
    --size 64 \              # 体素最大尺寸 (默认: 32, 最大: 256)
    --threshold 100 \        # 透明度阈值 (默认: 128)
    --color-source front     # 上色参考: front/right/top (默认: front)
```

## 三视图要求

### 图片格式
- 推荐 PNG 格式（支持透明通道）
- 也支持 JPG，但需要用纯白/纯黑背景区分实体和空白

### 透明度规则
- **alpha >= 阈值** → 视为实体（会放置体素）
- **alpha < 阈值** → 视为空白（不放体素）
- 如果图片没有透明通道（如 JPG），所有像素都会被视为实体

### 视图方向约定

```
正视图 (front)          右视图 (right)          顶视图 (top)
从前方看 (Z→)           从右方看 (X→)           从上方看 (Z→)
┌──────────┐            ┌──────────┐            ┌──────────┐
│          │            │          │            │          │
│  宽 = X  │            │  宽 = Y  │            │  宽 = X  │
│  高 = Z  │            │  高 = Z  │            │  高 = Y  │
│          │            │          │            │          │
└──────────┘            └──────────┘            └──────────┘
```

### 制作建议
1. 三张图的**高度应一致**（正视图和右视图共享 Z 轴高度）
2. 正视图的宽度应与顶视图的宽度一致（共享 X 轴）
3. 右视图的宽度应与顶视图的高度一致（共享 Y 轴）
4. 背景设为**全透明**，实体部分保持不透明
5. 轮廓尽量清晰，避免半透明边缘

## 工作原理

```
正视图 mask (XZ)  ∩  右视图 mask (YZ)  ∩  顶视图 mask (XY)  =  体素模型
```

三个视图分别定义了从三个方向看过去的"剪影"，取交集后得到三维体素数据。

## 后续处理

1. 用 MagicaVoxel 打开生成的 `.vox` 文件
2. 手动修正细节（自动生成的粗模可能有多余体素）
3. 调整颜色和材质
4. 导出为 `.obj` / `.glb` 等格式用于游戏引擎

## 常见问题

**Q: 生成的体素数量为 0？**
- 检查图片是否有透明通道，JPG 没有透明通道会导致所有像素都被视为实体
- 尝试降低 `--threshold` 值

**Q: 模型有很多多余的体素？**
- 这是投影交集算法的固有限制，凹面结构会产生多余体素
- 在 MagicaVoxel 中手动用橡皮擦清理

**Q: 颜色不对？**
- 尝试切换 `--color-source` 参数使用不同视图的颜色
- 在 MagicaVoxel 中手动重新上色效果更好
